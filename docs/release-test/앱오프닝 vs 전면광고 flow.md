`ad_policy` 테이블의 컬럼(Column)들을 우리의 규칙에 적용한 **최종 수정판 명세서**입니다.

### ⚠️ 전제 조건 (앱이 해야 할 일)

Supabase는 **"최대 몇 번까지 허용할지(Policy)"**만 알려줍니다.

실제로 **"내가 오늘 몇 번 보여줬는지(Count)"**는 앱이 스스로 노트(SharedPreferences)에 적어가며 기억해야 합니다.

---

### 규칙 1. 전면 광고 (Interstitial Ad)

**"나갈 때 보여주되, 사장님이 허락한 횟수까지만!"**

- **기본 타이밍:** [뒤로 가기] 버튼 클릭 시 (기존과 동일)
- **Supabase 검사 (Check List):**
    1. `is_active`가 `true`인가? (전체 광고 스위치)
    2. `ad_interstitial_enabled`가 `true`인가? (전면 광고 스위치)
    3. **[시간 제한]** 최근 1시간 동안 본 횟수 < `ad_interstitial_max_per_hour` (예: 3회) 인가?
    4. **[일일 제한]** 오늘 하루 동안 본 횟수 < `ad_interstitial_max_per_day` (예: 20회) 인가?
- **결과:**
    - 모두 **YES**면? 👉 광고 노출 ⭕ -> (보여준 뒤 앱 내부 카운트 +1 증가)
    - 하나라도 **NO**면? 👉 광고 패스 ❌ -> 바로 화면 이동

### 규칙 2. 앱 오프닝 광고 (App Open Ad)

**"켤 때 보여주되, 너무 자주는 안 됨!"**

- **기본 타이밍:** 앱 실행(스플래시) 또는 복귀(Background -> Foreground)
- **Supabase 검사 (Check List):**
    1. 현재 전면 광고가 떠 있는가? (`isShowingAd` 체크) 👉 **YES면 즉시 중단** (충돌 방지)
    2. `is_active`가 `true`인가?
    3. `ad_app_open_enabled`가 `true`인가?
    4. **[시간 제한]** 최근 1시간 동안 본 횟수 < `app_open_max_per_hour` (예: 2회) 인가?
    5. **[일일 제한]** 오늘 하루 동안 본 횟수 < `app_open_max_per_day` (예: 15회) 인가?
- **결과:**
    - 조건 만족 시 👉 광고 노출 ⭕ -> (보여준 뒤 앱 내부 카운트 +1 증가)

### 규칙 3. 배너 광고 (Banner Ad)

**"항상 떠 있지만, 끄라고 하면 꺼야 함"**

- **위치:** 화면 하단
- **Supabase 검사:**
    1. `is_active`가 `true`인가?
    2. `ad_banner_enabled`가 `true`인가?
- **결과:**
    - 둘 다 **YES**면 👉 배너 표시
    - 아니면 👉 배너 영역 자체를 숨김 (`View.GONE` 처리하여 화면 넓게 쓰기)

### [개발자가 구현해야 할 데이터 흐름도]

이제 코딩할 때 머릿속에 이 그림을 그리셔야 합니다.

1. **앱 실행 (Start)**
    - 가장 먼저 Supabase의 `ad_policy` 테이블에서 우리 앱(`app_id`)의 데이터를 가져옵니다.
    - 이 데이터를 **앱 내부 저장소(Singleton 객체)**에 잠시 보관합니다. (예: `AdPolicyManager`)
2. **광고 기회 발생 (Trigger)**
    - 예: 사용자가 뒤로 가기를 눌렀다!
3. **판사 등장 (Logic)**
    - 앱: "지금 광고 보여줘도 되나요?"
    - 판사: "잠깐, 장부를 보자."
        - Supabase 규칙: "시간당 3번까지만 된대."
        - 내 수첩(SharedPref): "너 아까 10분 전에 1번, 30분 전에 1번 봤어. 지금 2번 째네."
    - 판결: "3번 안 넘었으니 **승인!**"
4. **실행 및 기록 (Action)**
    - 광고를 띄웁니다.
    - 내 수첩에 적습니다. "지금 1번 더 봤음 (총 3회째)"
