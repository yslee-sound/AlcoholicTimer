## 📝 단위 테스트 시나리오 P6 **수정 요청 명세서**

### 1. 개요 및 수정 배경

현재 테스트 방침은 **Ad Policy 로드 실패 시 안전망**을 구축하는 것이 목표입니다. 기존의 '모두 비활성화(Fail-Safe)' 대신, **수익을 유지**하면서도 **AdMob 정책 위반을 최소화**하기 위한 **'안전한 기본 정책(Default Fallback Policy)'** 검증으로 시나리오를 수정합니다.

* **수정 대상:** 단위 테스트 시나리오 **P6** (정책 로드 실패 시 대처)
* **목표:** `AdPolicyRepository`가 정책 로드에 실패할 경우, 앱이 **오류 없이** 복구되어 **미리 정의된 안전한 기본 정책 객체**를 반환하는지 검증합니다.

***

### 2. 정의: 안전한 기본 정책 (Default Fallback Policy)

**정책 로드 실패 시 (Supabase 호출 실패, 빈 데이터, 파싱 오류 등) `AdManager`에 주입되어야 할 안전한 기본 정책의 구체적인 값**은 다음과 같습니다. 이 값들은 수익성을 유지하면서 정책 위반을 최소화하도록 설정됩니다.

| 필드 이름 (DB 컬럼) | 데이터 타입 | **Fallback 값** (Mock Data) | 정책 적용 이유 (수익/안전성) |
| :--- | :--- | :--- | :--- |
| **`is_active`** | Boolean | `true` | 수익 유지를 위해 전체 광고 활성화. |
| `ad_app_open_enabled` | Boolean | `true` | 앱오프닝 광고 활성화 허용. |
| `ad_interstitial_enabled` | Boolean | `true` | 전면 광고 활성화 허용. |
| `ad_banner_enabled` | Boolean | `true` | 배너 광고 활성화 허용. |
| `ad_interstitial_max_per_day` | Int | **5** | 일일 한도를 **보수적으로 낮게** 설정하여 과도한 노출 방지. |
| `app_open_cooldown_seconds` | Int | **300 (5분)** | 쿨다운 시간을 **매우 길게** 설정하여 앱오프닝 광고 중첩 및 정책 위반 위험 최소화. |
| `app_open_max_per_hour` | Int | **1** | 시간당 한도를 **최소로** 설정하여 과도한 노출 방지. |

### 4. 개발팀 요청 사항 (P6 구현 시 유의점)

* **테스트 검증:** 단위 테스트 시나리오 P6에서는 `AdPolicyRepository`가 반환한 객체의 **`app_open_cooldown_seconds`가 300인지, `ad_interstitial_max_per_day`가 5인지 등** **핵심 Fallback 값이 정확한지** `assertEquals`를 사용하여 검증해야 합니다.
