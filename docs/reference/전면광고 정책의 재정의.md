# 📄 [AlcoholicTimer] 광고 운영 및 수익화 정책서

작성일: 2025. 12. 04

작성자: 안드로이드 앱 수익화 전문가 (Gemini)

대상 앱: AlcoholicTimer (금주 타이머 & 기록)

---

## 1. 핵심 전략 요약

- **목표:** 사용자 경험(UX)을 해치지 않으면서 수익을 창출하는 '지속 가능한 수익화'.
- **주요 광고 형식:** 전면 광고 (Interstitial), 앱 오프닝 (App Open).
- **핵심 원칙:** **"모든 전면형 광고는 하나의 통합 쿨타임(5분)을 공유한다."**

---

## 2. 광고 배치 및 노출 시점 (Trigger)

### A. 전면 광고 (Interstitial)

- **노출 위치:** 작업이 '완료'되고 화면이 전환되는 순간.
- **구체적 적용 사례 (설정 화면):**
    1. 사용자가 **[적용하고 절약 금액 확인하기 >]** 버튼 클릭.
    2. (쿨타임 체크 통과 시) **전면 광고** 노출.
    3. 광고 닫기(X) 후 -> **결과 화면/메인 화면**으로 이동.
- **기술적 요구사항:** 화면 진입 시 광고를 미리 로드(Pre-load)하여, 클릭 시 즉시(Show) 보여줄 것.

### B. 앱 오프닝 (App Open)

- **노출 위치:** 앱이 실행되거나(Cold Start), 백그라운드에서 포그라운드로 돌아올 때.
- **특이사항:** 앱 오프닝 광고도 전면 광고와 **동일한 쿨타임 규칙**을 따름.

---

## 3. 쿨타임(빈도) 정책 : [5분 통합 룰]

### A. 정책 정의

- **규칙:** 사용자가 마지막으로 전면형 광고(앱 오프닝 포함)를 본 지 **5분(300초)**이 지나지 않았다면, 어떤 위치에서도 광고를 노출하지 않음.
- **범위:** 앱 내 모든 전면 광고 단위 + 앱 오프닝 광고 단위가 하나의 타이머를 공유.

### B. AdMob 콘솔 설정값 (Server Side)

- **설정 메뉴:** [앱 설정] -> [빈도 게재 설정]
- **설정 수준:** **앱 수준 (App Level)** 적용.
- **값:** 사용자당 **5분**에 **1회** 노출.
    - *효과: 앱 오프닝과 전면 광고가 서로 횟수 제한을 공유하여 '광고 폭탄' 방지.*

### C. 앱 내부 로직 (Client Side)

- **변수:** `last_ad_show_time` (마지막 광고 노출 시간, SharedPreference 저장 권장).
- **판단 로직:**
    - `IF (현재시간 - last_ad_show_time > 5분)` -> **광고 노출 (Show)**
    - `ELSE` -> **광고 스킵 (기능 바로 실행)**

---

## 4. 기술 구현 가이드 (개발자 필독)

### A. 타이머 리셋 시점 (중요 ⭐)

- **절대 원칙:** 단순히 광고를 요청(Load)했을 때가 아니라, **실제로 화면에 떴을 때(Show)**만 시간을 갱신한다.
- **코드 위치:**Kotlin

  # 

  `// AdMob 콜백 함수 내부
  override fun onAdShowedFullScreenContent() {
      // 광고가 사용자 눈앞에 떴을 때만 실행
      saveLastAdTime(System.currentTimeMillis()) 
  }`


### B. 비상 스위치 (Kill Switch)

- **도구:** Firebase Remote Config.
- **기능:** `is_ad_enabled` (Boolean) 값을 생성하여, `False`일 경우 앱 내 모든 광고 요청 코드를 실행하지 않음.
- **용도:** 심사 제출 시, 긴급 버그 발생 시 즉시 광고 차단.
